About this step

          

            

              
画面分割の発想

              

                これまで盤面 = キャンバス全域でしたが、ここで「右側の 160 px」を UI 領域として
                分離します。盤面描画のループは今まで通り x = col * CELL で 0〜399 に収まり、
                パネルは x = 414 以降。自然に住み分けできます。
              

            


            

              
countStones と集計の型

              

                countStones は「盤面を数える」典型的な集計処理。
                カウンタ変数を 0 で初期化し、for で走査し、条件に合えば count++、最後に return。
                今後、isGameOverByFill や finalScore などを足す時にも
                同じパターンを再利用します。
              

            


            

              
動かしてみる

              

                初期配置では BLACK 2、WHITE 2、TURN = Black と表示されるはず。
                着手するたびに数が変わって、手番も Black ↔ White と入れ替わります。
